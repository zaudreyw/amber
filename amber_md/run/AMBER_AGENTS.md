## What is Amber MD?

**Amber** (Assisted Model Building with Energy Refinement) is a suite of biomolecular simulation programs for molecular dynamics and related tasks.

### Core Characteristics

- **Platform**: HPC-ready with CPU (`sander`, `pmemd`) and GPU (`pmemd.cuda`) support
- **Domain**: Biomolecular simulation — proteins, nucleic acids, lipids, small molecules
- **Force Fields**: ff14SB, ff19SB, GAFF/GAFF2, OPC/TIP3P water models
- **Repository**: https://github.com/amber-md
- **Documentation**: https://ambermd.org/Manuals.php
- **License**: Commercial (Amber) / Free (AmberTools)

### Primary Programs

| Program      | Description                                        |
| ------------ | -------------------------------------------------- |
| `sander`     | General MD engine; reference implementation        |
| `pmemd`      | High-performance MD engine                         |
| `pmemd.cuda` | GPU-accelerated MD                                 |
| `tleap`      | System preparation: topology + coordinate files    |
| `cpptraj`    | Trajectory analysis                                |
| `antechamber`| Small molecule parameter generation (GAFF)         |

## Required Input Files

A complete Amber MD setup requires these files:

| File | Role | How produced |
|------|------|--------------|
| `tleap.in` | LEaP script: build topology + coordinates | **You write this** |
| `mdin_min.in` | Minimization control file | **You write this** |
| `mdin_heat.in` | Heating (NVT) control file | **You write this** |
| `mdin_equil.in` | Equilibration (NPT) control file | **You write this** |
| `mdin_prod.in` | Production MD control file | **You write this** |

Write `tleap.in` and all `mdin_*.in` files. The `.prmtop` and `.inpcrd` files are generated at runtime by running `tleap -f tleap.in`.

## mdin File Format

Amber `.mdin` files use Fortran NAMELIST format. The structure is:

```
Title line (required; one line; free text)
 &cntrl
  key1=value1,
  key2=value2,
  ...
 /
```

### Rules

- The first line is a free-text title (one line only).
- The `&cntrl` namelist block must be present.
- Each parameter is `key=value,` — the trailing comma is required except on the last entry.
- The closing ` /` (space + slash) must be on its own line.
- White space around `=` is optional but conventional.
- Integer and real values do NOT need quotes; strings need single quotes.

### Optional Additional Namelists

```
 &ewald
  dsum_tol=0.000001,
 /
 &wt
  type='END',
 /
```

## tleap Script Format

`tleap.in` is a LEaP command script. Common commands:

```bash
# Load force field parameters
source leaprc.protein.ff19SB
source leaprc.water.opc

# Load molecule from PDB
mol = loadpdb protein.pdb

# Add counterions
addions mol Na+ 0
addions mol Cl- 0

# Solvate with water box (10 Å padding)
solvateBox mol OPCBOX 10.0

# Save topology and coordinates
saveamberparm mol system.prmtop system.inpcrd

# Inspect the system
check mol

quit
```

### Common Force Field Sources

| Command | Force Field |
|---------|------------|
| `source leaprc.protein.ff19SB` | Proteins (recommended) |
| `source leaprc.protein.ff14SB` | Proteins (older) |
| `source leaprc.DNA.OL15` | DNA |
| `source leaprc.RNA.OL3` | RNA |
| `source leaprc.water.opc` | OPC water model |
| `source leaprc.water.tip3p` | TIP3P water model |
| `source leaprc.gaff2` | Small molecules (GAFF2) |

### Common Water Box Commands

| Command | Box type |
|---------|----------|
| `solvateBox mol OPCBOX 10.0` | OPC water, 10 Å buffer |
| `solvateBox mol TIP3PBOX 10.0` | TIP3P water, 10 Å buffer |
| `solvateOct mol OPCBOX 10.0` | OPC, truncated octahedron |

## Common Simulation Workflows

### 1. Standard Protein MD (4-stage workflow)

**Stage 1 — Energy minimization** (`mdin_min.in`):
```
Minimization
 &cntrl
  imin=1,
  maxcyc=2000,
  ncyc=1000,
  ntpr=100,
  ntwx=0,
  cut=10.0,
  ntb=1,
  igb=0,
 /
```

**Stage 2 — Heating NVT** (`mdin_heat.in`):
```
Heating
 &cntrl
  imin=0,
  irest=0,
  ntx=1,
  nstlim=25000,
  dt=0.002,
  ntc=2,
  ntf=2,
  cut=10.0,
  ntb=1,
  ntp=0,
  ntt=3,
  gamma_ln=1.0,
  tempi=0.0,
  temp0=300.0,
  ntpr=500,
  ntwx=500,
  ntwr=5000,
  ig=-1,
  nmropt=1,
 /
 &wt
  type='TEMP0',
  istep1=0, istep2=25000,
  value1=0.0, value2=300.0,
 /
 &wt
  type='END',
 /
```

**Stage 3 — Equilibration NPT** (`mdin_equil.in`):
```
Equilibration
 &cntrl
  imin=0,
  irest=1,
  ntx=5,
  nstlim=50000,
  dt=0.002,
  ntc=2,
  ntf=2,
  cut=10.0,
  ntb=2,
  ntp=1,
  barostat=2,
  pres0=1.0,
  taup=2.0,
  ntt=3,
  gamma_ln=1.0,
  temp0=300.0,
  ntpr=500,
  ntwx=500,
  ntwr=5000,
  ig=-1,
 /
```

**Stage 4 — Production NPT** (`mdin_prod.in`):
```
Production MD
 &cntrl
  imin=0,
  irest=1,
  ntx=5,
  nstlim=500000,
  dt=0.002,
  ntc=2,
  ntf=2,
  cut=10.0,
  ntb=2,
  ntp=1,
  barostat=2,
  pres0=1.0,
  taup=2.0,
  ntt=3,
  gamma_ln=1.0,
  temp0=300.0,
  ntpr=1000,
  ntwx=1000,
  ntwr=10000,
  ig=-1,
 /
```

## Common Namelist Parameters

### Simulation Control (`&cntrl`)

| Parameter | Values | Description |
|-----------|--------|-------------|
| `imin` | 0=MD, 1=minimization | Run type |
| `irest` | 0=new, 1=restart | Restart from prior run |
| `ntx` | 1=coords, 5=coords+vel | Input format |
| `nstlim` | integer | Number of MD steps |
| `dt` | 0.001–0.002 | Timestep (ps) |
| `ntc` | 1=no SHAKE, 2=H bonds, 3=all | SHAKE constraints |
| `ntf` | 1=all forces, 2=no H-bond forces | Force calculation |
| `cut` | real (Å) | Non-bonded cutoff; 999 for no cutoff |
| `ntb` | 0=no PBC, 1=const V, 2=const P | Periodic boundary |
| `ntp` | 0=const V, 1=isotropic, 3=anisotropic | Pressure control |
| `ntt` | 0=const E, 1=Berendsen, 3=Langevin | Thermostat |
| `temp0` | real (K) | Target temperature |
| `tempi` | real (K) | Initial temperature (heating) |
| `gamma_ln` | real (ps⁻¹) | Langevin collision frequency |
| `ig` | -1=random, integer=seed | Random seed |
| `igb` | 0=explicit, 1/2/5/7/8=GB model | Solvation model |
| `ntpr` | integer | Print frequency (mdout) |
| `ntwx` | integer | Trajectory write frequency |
| `ntwr` | integer | Restart write frequency |
| `maxcyc` | integer | Max minimization cycles |
| `ncyc` | integer | Steepest-descent to conjugate-gradient switch |

### Pressure Coupling

| Parameter | Values | Description |
|-----------|--------|-------------|
| `barostat` | 1=Berendsen, 2=MC | Barostat type |
| `pres0` | 1.0 | Target pressure (bar) |
| `taup` | 1.0–2.0 | Pressure relaxation time (ps) |

### Restraints

| Parameter | Values | Description |
|-----------|--------|-------------|
| `ntr` | 0=off, 1=on | Positional restraints |
| `restraint_wt` | real | Restraint force constant (kcal/mol/Å²) |
| `restraintmask` | string | Atom mask for restrained atoms |

## Quick Tips for AI Agents

1. **Stage order matters**: Always write minimization → heating → equilibration → production. Restart files chain the stages (`irest=1`, `ntx=5`).

2. **SHAKE consistency**: If `ntc=2`, set `ntf=2` as well. If `ntc=1`, set `ntf=1`.

3. **PBC bookkeeping**: `ntb=1` → constant volume (NVT); `ntb=2` + `ntp=1` → constant pressure (NPT). Implicit solvent uses `ntb=0`.

4. **Cutoff**: For explicit solvent use `cut=10.0` (or 8.0–12.0 range). For implicit/vacuum use `cut=999.0`.

5. **Timestep**: 2 fs (`dt=0.002`) is standard with SHAKE (`ntc=2`). Use 1 fs (`dt=0.001`) without SHAKE.

6. **Random seed**: Always use `ig=-1` for production runs (new random seed each run).

7. **tleap force field matching**: The force field sourced in `tleap.in` must be compatible with the `mdin` settings (e.g., OPC water needs `OPCBOX` in `solvateBox`).

8. **File naming convention**: Use stage-prefixed names — `mdin_min.in`, `mdin_heat.in`, `mdin_equil.in`, `mdin_prod.in` — to make the pipeline self-documenting.

9. **Check system before saving**: Always include `check mol` before `saveamberparm` in the tleap script to catch valence/charge issues.

10. **Output files to specify**: `mdin` → `mdout`; trajectory → `mdcrd`; restart → `restrt`; reference → `refc`. Conventions: `output_min.mdout`, `traj_prod.nc`, etc.
