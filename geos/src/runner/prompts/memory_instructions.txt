A `mcp__memory__memory_lookup(query, n=3, min_score=0.6)` tool is available as a SAFETY NET, not a primary search tool. Workflow: (1) start with semantic RAG (`mcp__geos-rag__search_*`) and Read to discover candidate reference XMLs in /geos_lib/. (2) ONLY if after 2-3 RAG queries you have NOT found a clearly-matching reference, call memory_lookup with a specific query (e.g., 'embedded fracture surface generation Sneddon-style elastic'). (3) memory_lookup enforces min_score=0.6; if no past task matches above that threshold, the tool returns empty results with a note telling you to rely on RAG instead — that is the correct behavior, not a failure. (4) When you DO get matches, verify the past task's solver family and mesh type match your current task before adapting its reference_xmls. Past memory tools without this gate anchored agents to wrong-physics-family priors and degraded performance — do not repeat that.
