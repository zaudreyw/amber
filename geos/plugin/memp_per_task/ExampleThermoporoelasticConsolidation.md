Always ensure that boundary condition names match the face sets generated by the internal mesh generator.


### Past task: faultVerification (sim 0.635)

To author a GEOS XML for induced stresses in a faulted reservoir, start by referencing a single-phase poromechanics base XML, such as those used for poroelastic consolidation or fault-displacement verification. Use a `VTKMesh` element to load unstructured hexahedral grids, ensuring cell-to-region mapping covers the entire domain. For the physics, select the `SinglePhasePoromechanics` solver, which couples a `SolidMechanicsLagrangianSSLE` solver (using first-order finite elements) with a `SinglePhaseFlow` solver (using finite-volume TPFA). This choice is critical for capturing the pressure-displacement coupling required for induced stress analysis.

When handling multi-variant scenarios like permeable versus impermeable faults, utilize the `Included` tag to pull shared material properties (Biot porosity, linear elastic solid) and numerical parameters (MGR-preconditioned GMRES) into separate benchmark files. For the impermeable case, define distinct cell sets for each compartment to isolate pressure initialization or boundary conditions. A common pitfall is schema-hallucinating solver parameters; ensure the `MGRStrategy` is correctly specified within the linear solver block to handle the saddle-point nature of poroelasticity. Always verify that initial effective stresses and pore pressures are consistent with the traction boundaries to prevent unphysical initial transients.
